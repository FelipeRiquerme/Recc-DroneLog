# DroneLog - Avaliação de Recuperação

## Tema

A empresa fictícia DroneLog registra operações de mapeamento, inspeção e monitoramento com drones. Com cada missão vinculada a um piloto e a um drone. 
O desenvolvedor contratado para criar essa API para controle de operações com drones em campo passou à você a responsabilidade de completar o projeto em andamento.

Ele disse que a aplicação deve permitir o cadastro e a consulta de:

- drones utilizados nas missões/operações,
- pilotos responsáveis pelos voos,
- missões de voo realizadas ou planejadas.

O projeto possui partes prontas e partes incompletas. Os pontos de atenção estão marcados com `TODO` dentro do código.

## Entrega

Você deve entregar um repositório público no GitHub contendo o projeto funcionando, organizado e com commits próprios, seguindo os critérios desta avaliação.

## Consulta

A pasta `/docs` contém materiais de apoio. Os documentos servem como consulta, mas não indicam exatamente a solução de cada arquivo.

## Tecnologias esperadas

- Java
- Spring Boot
- Spring Web
- Spring Data JPA
- PostgreSQL
- Bean Validation
- Swagger/OpenAPI
