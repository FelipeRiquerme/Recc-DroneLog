# Instruções gerais

## Contexto

Você recebeu um projeto base de uma API chamada **DroneLog**, voltada ao controle simples de operações com drones em campo.

A aplicação trabalha com três recursos principais:

- **Drone**;
- **Piloto**;
- **Missão de Voo**.

Cada arquivo pode estar em um nível diferente de desenvolvimento. Algumas partes estão mais completas, outras estão parcialmente feitas e outras exigem maior autonomia.

## Mínimo esperado

Para que a recuperação seja considerada entregue, o projeto deve:

- iniciar sem erro impeditivo;
- conectar ao PostgreSQL;
- possuir CRUD funcional para os recursos principais;
- usar camadas de forma organizada;
- usar DTOs quando necessário;
- aplicar validações básicas;
- disponibilizar a documentação pelo Swagger;
- estar em repositório público no GitHub.

## Orientações

1. Analise primeiro a estrutura do projeto.
2. Verifique os arquivos que já possuem uma implementação mais completa.
3. Use os TODOs do código como pontos de atenção.
4. Consulte os materiais da pasta `/docs` para revisar conceitos.
5. Corrija, complete e teste uma parte por vez.

## Consulta

Durante a atividade, utilize os materiais disponibilizados na pasta `/docs` como apoio.

Os documentos não apresentam a solução final. Eles servem para relembrar estrutura, responsabilidades das camadas, validações, relacionamentos, filtros, Swagger e erros comuns.

## Entrega

A entrega deve ser feita por meio de um repositório público no GitHub. O link do repositório deve ser enviado no AVA.
